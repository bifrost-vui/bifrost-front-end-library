# Drupal implementation
You have a **Drupal installation** and you want to use Bifrost Design System.

<br/><br/>

## 1. Install NPM Package
In your theme:

```bash
npm install @videotron-ui/bifrost-front-end-library@<specific-version> --save

# Or the latest version
npm install @videotron-ui/bifrost-front-end-library@latest --save
```


## 2. Add Namespace and filters
Install [components module](https://www.drupal.org/project/components) in Drupal.
Then, add theses paths to **your-theme.info.yml**:
```
component-libraries:
  [...]
  bf-components:
    paths:
      - node_modules/@videotron-ui/bifrost-front-end-library/source/components/
  bf-utils:
    paths:
      - node_modules/@videotron-ui/bifrost-front-end-library/source/utils/
```

### Add filters list
See [Filters list](?path=/docs/implementation-drupal-filters--page)

## 3. Add Base JS

Replace `uikit-marvel.js` by `bifrost.js` from the NPM Package:
`node_modules/@videotron-ui/bifrost-front-end-library/assets/js/bifrost.js`.



## 4. Add Base style

Replace `uikit-marvel.css` by `bifrost.css` from the NPM Package, in order to provide the same CSS base for all Videotron websites.
Here 4 possibles options:

<br/>

### Option 1:
Create a library **your-theme.libraries.yml**:
```js
global-styling:
  css:
    theme:
        styles.css: {}
        'node_modules/@videotron-ui/bifrost-front-end-library/assets/css/bifrost.css': {}
```

Then, add this library to **your-theme.info.yml**:
```js
libraries:
  - your-theme/global-styling
```

### Option 2:
In your **CSS file**:
```css
@import 'node_modules/@videotron-ui/bifrost-front-end-library/assets/css/bifrost.css';

/* Or if you are using SASS: */
@import '~@videotron-ui/bifrost-front-end-library/assets/css/bifrost.css';
```

### Option 3:
In the **head** of your pages:
```html
<head>
    <link href="./node_modules/@videotron-ui/bifrost-front-end-library/assets/css/bifrost.css" rel='stylesheet' />
</head>
```

## 5. Migrate a Component
- All Bifrost components are visible on this Storybook. In the sidebar, check if your component is available:

<img src={imageStorybookCompnentList} style={{maxWidth: '800px'}} />

- Select the **Docs Tab** at the top of the component's page and look at the **Source Code** section.
- Please, take note that we improved the path to the component. So for example:
```jsx
{% include '@atoms/02-buttons/10-button-outline.twig' %}

# becomes

{% include '@bf-components/button/twig/index.twig' with {
    outline : true
} only %}
```
- In the **Component API** section, check that you send the right props to your component.


## 6. Themes
On Storybook, check if the component is available in a specific theme.
Wrap the component with a `data-theme='x'`.
```html
<div data-theme='business'>
    {% include '@bf-components/button/twig/index.twig' %}
</div>
```


## 7. Reversed style

#### What is reversed?
In every theme, we have a default background color. For exampe, in Videotron theme, the default is white. But some sections have black background. Theses sections are `reversed`.
//TODO: Add a link to Zeroheight Documentation.

> **Note:** We didn't call it white-mode/dark-mode in order to be "color agnostic" and then more futur-proof.

#### How to implement?
- Add a class `.reversed` in any HTML tag.
```jsx
// Note that if you want to display the reversed background,
// you need to add it independently.
<div class='reversed bf-color-bg-ground'>
    {% include '@bf-components/button/twig/index.twig' %}
</div>
```

- Some special components need to know explicitly the reversed state: `Navigation Header` and `Logo`.
Then you need to pass the `reversed` property to the include:
```jsx
{% include '@bf-components/navigation-header/twig/index.twig' with {
    reversed : true
} only %}
```
