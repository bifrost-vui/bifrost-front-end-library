# Icon Migration
> New icons set are available since **Bifrost v1.0.4**.

## New Icon Component
When you are using Bifrost Icon component.
- `type` => Not used anymore to get the icon. Now optionnal.
- `size` => `small` by default when icon is interface type, and `medium` by default for type illustration.

<br/>

## Get Official icon lists
You should rely on these paths to get the official and versionned icons lists:
- <a href='icons/list/illustrations.json' target='_blank'>
    Illustration: /icons/list/illustrations.json
</a>
<br/>
- <a href='icons/list/interfaces.json' target='_blank'>
    Interface: /icons/list/interfaces.json
</a>


If you want the flat list (without categories), you could use this `map`:
```jsx
import illustrationIcons from '../icons/list/illustrations.json';
const illustrationIconsFlatten = illustrationIcons.map(a => a.list).flat().sort();

```

With the `name`, you will able to get the svg file name with this pattern:
` {{ name ~ '-' ~ 'sm | lg' ~ '.svg' }}`

All SVG files have been updated and now exist in **large** and **small sizes** (naming convention: `filename-lg.svg` & `filename-sm.svg`).
Every **Icon component** now comes in 4 possible **sizes** : **large**, **medium**, **small** and **xsmall**.
