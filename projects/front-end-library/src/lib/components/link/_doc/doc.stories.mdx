# Link
    
Le **Lien** est un élément de navigation qui permet aux utilisateurs de cliquer pour accéder à une autre page ou à une autre section de la même page. Il peut être composé d'un texte ou d'une image, et peut être présenté sous différentes formes (paragraphe, bouton, etc.).

## **Utilisation**

### **Quand utiliser ce composant :**

- Utiliser le **Lien** lorsqu'il est nécessaire de fournir un moyen de navigation pour les utilisateurs.
- Utiliser le **Lien** lorsqu'il est important de permettre aux utilisateurs d'accéder à du contenu ou à des fonctionnalités supplémentaires.
- Utiliser le **Lien** pour des sites ou des applications à structure de navigation complexe.

### **Quand ne pas utiliser ce composant :**

- Ne pas utiliser le **Lien** lorsqu'il n'est pas nécessaire de fournir un moyen de navigation pour les utilisateurs.
- Ne pas utiliser le **Lien** lorsque la navigation est très simple et ne nécessite pas de lien supplémentaire.
- Ne pas utiliser le **Lien** si la taille de l'écran ne permet pas de l'afficher de manière claire et visible pour l'utilisateur.

## **Bonnes pratiques**

### **Faire :**

- Utiliser un texte clair et concis pour le **Lien**, qui reflète l'emplacement ou le contenu vers lequel il mène
- Mettre en évidence le **Lien** en utilisant une couleur de texte ou de fond différente
- Limiter le nombre de liens dans le contenu pour éviter de surcharger l'interface utilisateur

### **Ne pas faire :**

- Utiliser un texte trop long ou ambigu pour le **Lien**
- Mettre en évidence le **Lien** de manière trop intrusive
- Afficher un nombre excessif de liens dans le contenu, ce qui peut surcharger l'interface utilisateur

## **Accessibilité**

- Utiliser des couleurs de contraste suffisant pour le texte et le fond du **Lien**
- Utiliser des tags HTML appropriés pour le **Lien**, tels que **`<a>`**
- S'assurer que le focus du curseur est visible lorsque l'utilisateur navigue avec le clavier entre les liens
- Utiliser les attributs **`aria`** pour ajouter des informations supplémentaires sur le **Lien** aux technologies d'assistance, comme **`aria-label`** pour fournir un libellé alternatif et **`aria-current`** pour indiquer le lien actuel.

## **Rédaction**

- Les noms de page utilisés pour chaque lien doivent être clairs et concis, et doivent refléter l'emplacement ou le contenu vers lequel il mène.
- Évitez d'utiliser des noms de page trop longs ou ambigus pour chaque lien. Utilisez plutôt des termes compréhensibles par tous les utilisateurs.
- Si le **Lien** contient des icônes ou des images, assurez-vous de fournir un texte alternatif pour les utilisateurs qui ne peuvent pas voir l'image ou qui utilisent des technologies d'assistance.

## Drupal implementation example

```jsx
// Link native (default)
{% include '@bf-components/link/twig/index.twig' with {
    label       : 'Link native',
} only %}

// Link primary
{% include '@bf-components/link/twig/index.twig' with {
    label       : 'Link primary',
    hierarchy   : 'primary'
} only %}

// Or use .bf-link class
<a href='/' class='bf-link'>Link native</a>
<a href='/' class='bf-link bf-link--primary'>Link primary</a>
<a href='/' class='bf-link disabled' aria-disabled='true' role='link'>Link disabled</a>
```
