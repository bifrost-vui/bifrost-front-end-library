# Merge checklist

<br/>

> Each component should pass this checklist, else log a ticket in JIRA in order to improve the component later.

**Code structure**
- [ ] Component name and Component API are agnostic
- [ ] Component API is consistent with other components
- [ ] Accessible (use [Axe](https://www.deque.com/axe/) inside Storybook panel or [Pa11y](https://github.com/pa11y/pa11y) in the CI)
- [ ] Content resilient
- [ ] Responsive

**Styles**
- [ ] Use CSS variables
- [ ] Display correctly in reversed context
- [ ] Themable

**Tests**
- [ ] Validate visual regression
- [ ] Validate in supported browser versions (run `npx browserslist` to list all browsers supported)
- [ ] Unit test (if relevant)

**Review**
- [ ] Validate by designers/stakeholders

**Documentation**
- [ ] Document component API
- [ ] Document an integration example for Drupal or Angular.
- [ ] Display Figma Panel
