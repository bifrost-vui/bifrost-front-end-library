# Angular implementation
You have an **Angular installation** and you want to use Bifrost Design System.

<br/><br/>

## Requirements
- **Angular** version X or superior
- ...

## Architecture _(Migration)_
Structure your application to embrace Design System abstraction.
> Angular article: [Smart components vs Presentational components](https://blog.angular-university.io/angular-2-smart-components-vs-presentation-components-whats-the-difference-when-to-use-each-and-why/)

<br/>

## 1. Install the package
_TODO: Add Link to changelog version on Bitbucket._
```Shell
npm install bifrost-front-end-library --save
```


## 2. Include components
Import component in **app.module.ts**:

```ts
import { Button } from 'bifrost-front-end-library';

@NgModule({
  declarations: [],
  imports: [
    //...
    Button
  ],
  providers: [],
  bootstrap: []
})
export class AppModule { }
```

Use it in your template (note that all components tags are prefix with **bi-**).
```html
<bi-button styleClass='override' label='button from design system'></bi-button>
```

## 3. Add a base styles
Add **base.min.css** in **angular.json**:
```ts
"styles": [
  "node_modules/bifrost-front-end-library/base.min.css",
  //...
],
```

**API documentation is provided in Storybook:**
Select any component, you will see tabs at the very top:
- **Docs tab** displays complete documentation with all component variations in a single page. 

    > By default, we display **Angular components** unless you see the title **Drupal**.

- **Canvas tab**: Please, select **Controls panel** and explore the (interactive) API documentation.

## 4. _Optional:_ Access global SASS variables
Furthermore, you will be able to import variables in your sass files:
```css
@import 'node_modules/bifrost-front-end-library/source/globals/_variables.scss'

body {
    background-color: $primary;
}
```
