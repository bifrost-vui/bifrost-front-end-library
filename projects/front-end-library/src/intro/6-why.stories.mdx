# Why a Design System?

<br/><br/>

### One Source of truth for all
- Ensure consistency across products.
- Design is always alive. Let's embrase this.
- Could contains Brand, marketing assets, ...

<br/>

### Ensure strong components
##### For everyone:
- A single entry point to share and validate with stakeholders
##### For dev and designer
- Component-Driven Development
- Visual is a designer responsability: CI pipeline with accessibility and visual regressions
- Manual testing: browser/OS compatibility, responsive, ...

<br/>

### Same definitions for Developers and Designers
- Ensure consistency between Figma and Production

<br/>
